Name Description Service users? Local users?
POSIX Integrates with any compatible POSIX based distributed file-system. This integration uses local users.
UCloud Integrates with any compatible POSIX based distributed file-system. This integration uses a single service user for all files. ACLs are only managed through UCloud. This makes it suitable, for example, with container-like job orchestrators.

POSIX

This plugin requires a matching file plugin. These two plugins must have the same name and match the same set of products.

The extension script to run to locate the drives (file collections) of an end-user. The script runs in the context of the UCloud/IM service account and will run immediately after a resource allocation has been granted to a workspace.

Example request (personal workspace): { "type": "user", "username": "UCLOUD USERNAME", "uid": 100000 } Example request (project): { "type": "project", "projectId": "UCLOUD PROJECT ID", "gid": 410000 } Example response: [ { "path": "LOCAL PATH ON IM SERVER", "title": "TITLE WHICH THE USER SEES" }, { "path": "SECOND LOCAL PATH ON IM SERVER", "title": "SECOND TITLE WHICH THE USER SEES" }, { "path": "THIRD LOCAL PATH ON IM SERVER", "title": "THIRD TITLE WHICH THE USER SEES" } ]

The extension script to fetch the usage of a particular drive. The script runs in the context of the UCloud/IM service account and will run periodically. This script does not run in the context of the end-user.

The script will receive the path of one of the drives which has been returned by the driveLocator extension script. Each invocation should return the number of bytes used for that particular drive.

If this script is not defined, then no accounting will be reported to UCloud/Core. Any quotas applied directly on the file-system still apply.

Example request: { "path": "/home/myuser" } Example response: { "bytesUsed": 10000000000 }
